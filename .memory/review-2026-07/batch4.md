## Purpose Statements (criterion 1 — all pass)

- **init-project**: Scaffold crew-research workspace conventions (.memory, .scratch, AGENTS.md, .references) in a new or unstructured project.
- **adopt-project**: Migrate an existing brownfield project to crew-research conventions while preserving its custom rules and workflows.
- **project-audit**: Read-only drift check that deployed steering, skills, AGENTS.md, and CONTEXT.md still match project reality.
- **project-cleanup**: Actively consolidate project artifacts — promote scratch, process decisions to ADRs, dedupe memory, prune scripts and stale config.
- **project-winddown**: Extract portable lessons from a completed/paused project into a summary/ directory via parallel subagent review.
- **docs-audit**: Score project documentation against the D.O.C.S. anti-pattern framework and README maturity model.
- **tutorial-authoring**: Author hands-on tutorials with a spine/reference split that take readers from zero to a verified working result.
