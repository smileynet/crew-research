# Pipeline Routing by Style

Different styles benefit from different generation approaches. Route based on whether the style is voice-driven or structure-driven.

## Two Modes

### Voice-Heavy → Single-Pass Generation
The voice IS the structure. Let it flow freely within constraints.

**Characteristics**: distinctive narrator, rhythm-driven, confessional, atmospheric
**Examples**: Scorsese (confessional), Villeneuve (atmospheric), Miyazaki (gentle), Murakami (surreal), Gibson (sensory), King (conversational)

**Process**:
1. Load voice constraints + minimum elements
2. Generate in one pass
3. Revise with diagnostics after

### Structure-Heavy → Pipeline Generation (ending first)
The architecture carries the meaning. Build the skeleton before the flesh.

**Characteristics**: non-linear, puzzle-box, thematic architecture, parallel timelines
**Examples**: Nolan (temporal), Kubrick (geometric), Bong (class architecture), Coen (ironic structure), Crichton (systems), Sanderson (magic systems)

**Process**:
1. Write the ending/revelation first
2. Then write the setup that earns it
3. Revise for coherence

## Decision Test

Ask: "Does this style's power come from HOW it's told (voice) or HOW it's structured (architecture)?"

| Signal | Route |
|--------|-------|
| Distinctive narrator voice | Single-pass |
| Non-linear timeline | Pipeline (ending first) |
| Confessional/first-person | Single-pass |
| Puzzle-box or reveal structure | Pipeline (ending first) |
| Atmospheric/sensory | Single-pass |
| Parallel storylines that converge | Pipeline (ending first) |
| Stream of consciousness | Single-pass |
| Thematic architecture (symmetry, mirroring) | Pipeline (ending first) |

## Why This Matters

Voice-heavy styles lose energy when over-planned. The voice needs room to discover.
Structure-heavy styles collapse when generated linearly. The ending must exist before the setup can earn it.

Applying the wrong mode produces the most common failures:
- Voice style + pipeline = stilted, mechanical prose
- Structure style + single-pass = endings that don't land, setups that don't pay off
